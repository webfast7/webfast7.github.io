解题思路

假设当前已经存在一个数组如下:[1, 4, 2, 3, 4, 4],现在模的值为3,若当前存在一个子序列满足

推导过程：

模运算基本性质的推导

性质描述

对于整数 ( x ) 和 ( y )，若 ( x % k = y % k )，则 ( x - y ) 是 ( k ) 的整数倍（即 ( x - y = mk )，其中 ( m ) 为整数）。

推导过程

已知条件

给定：
[ (A + B) % k = (B + C) % k ]

应用模运算性质

根据上述性质，两数同余则差为 ( k ) 的倍数：
[ (A + B) - (B + C) = mk (m为整数) ]

化简表达式

化简左边的式子：
[ (A + B) - (B + C) = A - C ]

因此得到：
[ A - C = mk (即 A = C + mk) ]

证明 ( A % k = C % k )

模运算的定义：对于任意整数 ( x )，( x % k ) 表示 ( x ) 除以 ( k ) 的余数，记为 ( r )，满足：
[ 0 < r < k ]
且：
[ x = nk + r (n为整数) ]

证明步骤：

  1. 设 ( C % k = r )（即 ( C = nk + r )，( 0 leq r < k )）
  2. 代入 ( A = C + mk ) 得：
    [ A = (nk + r) + mk = (n + m)k + r ]
  3. 根据模运算定义，( A % k ) 即为 ( A ) 除以 ( k ) 的余数 ( r )，因此：
    [ A % k = r = C % k ]

题解思路

综上所述，我们证明了：
当[ (A + B) % k = (B + C) % k ]成立时， A % k = C % k也成立。

因此能连接的序列，其B数字旁边的其余两个数字的取模必然是相同的，则易发现最终得到的序列取模必然是a, b, a, b, a, b这种结构的

因此若当前k为3的时候，取模得到的值可能为0, 1, 2

则可能出现的序列即为[0, 0], [0, 1], [0, 2], [1, 0], [1, 1], [1, 2], [2, 0], [2, 1], [2, 2]

归并排序

算法原理

归并算法是对分治思想的实现。结合到算法中，一个无序的有理数序列，把它两两分成多个子序列，对这些子序列进行排序，此时多个子序列是有序的，然后再两两子序列结合，再次进行排序，持续进行这个过程，当前序列逐渐变得有序。这就是基本的算法思路。

编写思路

递归思路：既然当前是需要将无序序列分成多个子序列，那么定义出来一个左边界和一个右边界，然后在递归当前左边界和有边界，直到当前序列中只剩一个元素，则当前序列就是有序的，然后在对两两序列进行排序，逐渐向上排序，最后获得一个有序的序列。

快速排序

算法原理

通过取到一个标志位上的值，区分出小于这个值的部分和大于这个值的部分，再对两边进行排序。本质上来说，就是找出当前排序节点在这个序列中的位置。逐步的将当前大问题分为多个小问题去处理。

编写思路

既然上面已经明确了，当前我需要找出一个标准的点（pos值）去将当前的序列进行划分，那么不妨取当前序列的最后一个值（或中间值或左值均可）。当我对这一个值进行排序之后，那么这个值对于序列来说，这个index对应的值只可能是它或者相同值的数。那么，接下来需要面临的，此时我需要明确如何去递归的实现，即自上到下，从大的序列到小的序列逐渐缩小范围，最终得到的序列即为有效序列。
递归思路如下：

现在解决了当前的递归问题，剩下的就是如何去让当前一个序列分成大于pos值和小于pos值的两个区间。
那么我取最后一个值为pos值，现在让我的index停留在left-1的位置。此时，如果当前值大于pos的值，那么index就不能变动，因为index的值是后续pos需要交换的值。反之，小于pos时，此时交换当前值交换到index的位置。即如图所示。

当所有的交换完成之后，此时index对应的值以及前面的值，均为小于pos值的值，后面的值，均大于pos值。此时将当前的基准pos值与index+1的值进行交换，此时获得了当前pos值在序列中最终的位置。
得到的序列即为: 4 3 2 5 6 5 10 9 8 7

非递归思路：与上述类似，非递归写法的思路就是用栈去模拟当前的一个左边界和右边界，实现快速排序。
